# Chunked HTML Slide Generation

All readweave commands generate single-file HTML presentations. For presentations with more than ~20 slides, the HTML output exceeds what a single Write call can produce. This document defines the **Scaffold-Batch-Assemble** pattern that all commands use for reliable generation.

**Always use this pattern.** Even for smaller presentations, the overhead is minimal and it prevents failures on dense source material.

---

## Phase A: Write the Scaffold

Read the appropriate template file, replace placeholders, and write the file with **batch markers** inside `<div class="slides-wrapper">`. Batch markers are paired HTML comments that serve as insertion targets for Phase C.

### Scaffold structure

```html
<!-- ... template head, CSS, sidebar, nav ... -->

<div class="slides-wrapper">

<!-- BATCH-1: [description of slides in this batch] -->
<!-- /BATCH-1 -->

<!-- BATCH-2: [description of slides in this batch] -->
<!-- /BATCH-2 -->

<!-- BATCH-3: [description of slides in this batch] -->
<!-- /BATCH-3 -->

</div>

<!-- ... template JS ... -->
```

### Batch manifest

Include a **batch manifest** as an HTML comment immediately inside the slides-wrapper, before the first batch marker. This is the single source of truth for slide ordering:

```html
<div class="slides-wrapper">

<!--
BATCH MANIFEST
- Batch 1 (structural-opening): Title (slide 0), Overview (slide 1) [, Prerequisites (slide 2)]
- Batch 2 (content): EX-1 through EX-8, chapters 1-4
- Batch 3 (content): EX-9 through EX-16, chapters 5-8
- Batch 4 (content): EX-17 through EX-22, chapters 9-11
- Batch 5 (active-learning): Recall slides, Challenge slides
- Batch 6 (structural-closing): Discussion (slide N+1), Closing (slide N+2)
-->

<!-- BATCH-1: structural-opening — title, overview, prerequisites -->
<!-- /BATCH-1 -->

...
```

The descriptions inside batch markers must match the manifest. Each content batch lists the **exact excerpt IDs** (EX-N) it covers.

### Writing the scaffold

The scaffold itself (template + batch markers) is small — typically 20-30KB. Write it in a single Write call. Verify the file was created before proceeding to Phase B.

---

## Phase B: Generate Slide Batches

Launch **parallel agents** (one per batch) to generate the HTML for each batch of slides. Each agent writes its slides directly into the scaffold file.

### Batch sizing rules

| Slide type | Approx. size | Notes |
|---|---|---|
| Title slide | 0.8-1.2 KB | Fixed structure, small |
| Overview slide | 1.5-2.5 KB | Diagram complexity varies |
| Prerequisites slide | 1.0-1.5 KB | Fixed structure |
| Prose excerpt + margin notes | 1.5-2.5 KB | 2-3 margin notes typical |
| Code slide + margin notes | 2.0-3.5 KB | Code block + context + notes |
| Evolution/comparison slide | 2.0-3.0 KB | Side-by-side panels |
| Recall slide (6-8 cards) | 2.0-3.0 KB | Multiple Q&A pairs |
| Challenge slide (3-4 items) | 2.5-3.5 KB | Spec + hints |
| Discussion slide | 1.0-1.5 KB | 6-8 questions |
| Closing slide | 0.5-0.8 KB | Fixed structure |

**Maximum 8 content slides per batch.** At ~2.5KB average, 8 slides = ~20KB of HTML output — safely under the tool output limit.

**Structural batches** (opening, closing) are smaller and can be generated by the main context or by agents.

### Agent prompt template

Each batch agent receives this exact prompt structure. **Do not deviate from this format** — ambiguity causes agent loops.

```
You are generating HTML slides for a book presentation. Write the slides directly
into the scaffold file using the Edit tool.

**Your batch:** BATCH-N
**Slides to generate:** [list of slides with types]
**Excerpts to use:** [EX-IDs with their content from annotations.md]

**Instructions:**
1. Read the design guide at [path to read-design-guide.md or study-design-guide.md]
2. Read the current scaffold file at [path to output HTML file]
3. Generate the HTML for your assigned slides
4. Use the Edit tool to replace the content between <!-- BATCH-N: ... -->
   and <!-- /BATCH-N --> with your generated slides
5. The old_string for the Edit is exactly:
   <!-- BATCH-N: [description] -->
   <!-- /BATCH-N -->
   The new_string is:
   <!-- BATCH-N: [description] -->
   [your generated slide HTML here]
   <!-- /BATCH-N -->

**Output format rules:**
- Each slide is a <div class="slide ..."> block
- Follow the HTML patterns from the design guide exactly
- Use proper typographic entities (&mdash; &lsquo; &rsquo; &ldquo; &rdquo; &middot; &hellip;)
- Every slide needs a data-title attribute
- First slide of the ENTIRE presentation (only in Batch 1) gets class="active"
- Do NOT wrap output in markdown code fences
- Do NOT include any explanation — only the Edit tool call

**Excerpt content for your slides:**
[paste the relevant EX-N entries from annotations.md here]
```

### What each agent receives

1. **The design guide path** — HTML patterns to follow
2. **The scaffold file path** — where to insert slides
3. **The batch marker text** — exact old_string for the Edit
4. **The relevant annotations** — only the EX-N entries for this batch (not the full file)
5. **Slide type instructions** — what kind of slide each excerpt becomes

### Parallelism

Launch all batch agents in a **single message with multiple Agent tool calls**. They operate on different regions of the same file (different batch markers), so there are no conflicts.

**Exception:** If you're concerned about concurrent file edits, generate batches 2 at a time and wait for each pair to complete. The scaffold's batch markers ensure each Edit targets a unique string.

---

## Phase C: Verify Assembly

After all batch agents complete:

1. **Read the assembled file** to verify all batch markers have been filled (no empty `<!-- BATCH-N -->...<!-- /BATCH-N -->` pairs remain)
2. **Check slide count** — count `<div class="slide` occurrences, verify it matches the expected total
3. **Check first slide** — verify `class="slide active` appears exactly once, on the first slide
4. **Check closing tags** — verify the HTML is well-formed (every `<div class="slide` has a matching closing `</div>`)
5. **Check sidebar** — verify every slide has a `data-title` attribute

If any batch failed (agent error, empty output), re-run that specific batch agent. The batch markers make it safe to retry individual batches without affecting others.

## Command-Specific Batch Planning

### `master` (40-50 slides, 5-7 batches)

| Batch | Slides | Content |
|-------|--------|---------|
| 1 | 3 | Title, Overview (concept dependency diagram), Prerequisites |
| 2 | 7-8 | Prose excerpts + code slides, early chapters |
| 3 | 7-8 | Prose excerpts + code slides, middle chapters |
| 4 | 7-8 | Prose excerpts + code slides, late chapters |
| 5 | 4-5 | Remaining content slides + any overflow |
| 6 | 3-5 | Recall slides (2-3), Challenge slides (1-2) |
| 7 | 2-3 | Discussion, Closing |

Distribute prose excerpts and code slides **interleaved by chapter order**, not all prose then all code.

### `read` (25-35 slides, 3-5 batches)

| Batch | Slides | Content |
|-------|--------|---------|
| 1 | 2 | Title, Overview |
| 2 | 8-10 | Excerpt slides, first half |
| 3 | 8-10 | Excerpt slides, second half |
| 4 | 2-3 | Discussion, Closing |

For books with 30+ excerpts selected, add a 5th batch to keep each content batch at max 8 slides.

### `study` (30-65 slides, batches by book grouping)

| Batch | Content |
|-------|---------|
| 1 | Title, Library, Roadmap (structural opening) |
| 2-N | Content slides per thread or per 2-3 books (max 8 slides each) |
| N+1 | Reading Order, Discussion, Closing (structural closing) |

### `crawl` (20-55 slides, batches by thread)

| Batch | Content |
|-------|---------|
| 1 | Title, Map, structural opening |
| 2-N | One batch per thread (max 8 slides each) |
| N+1 | Discussion, Closing |

### `playlist` (15-65 slides, batches by video cluster)

| Batch | Content |
|-------|---------|
| 1 | Title, Arc Overview, structural opening |
| 2-N | One batch per video cluster (max 8 slides each) |
| N+1 | Discussion, Closing |

### `research` (55-85 slides, batches by learning stage)

| Batch | Content |
|-------|---------|
| 1 | Title, Source Library, Learning Path Roadmap (structural opening) |
| 2-N | One batch per learning stage (max 8 slides each, including stage transition + excerpts + evolution slides) |
| N+1 | Synthesis slides (cross-stage analysis) |
| N+2 | Bibliography, Discussion, Closing (structural closing) |

## Troubleshooting

**Agent produces empty output or loops:** The prompt was too vague. Ensure the agent prompt follows the template above exactly — especially the "Do NOT include any explanation" instruction.

**Edit tool fails (old_string not found):** Another batch agent already modified the file in a way that changed surrounding whitespace. Re-read the file to get the current exact text of the batch marker, then retry.

**Output still exceeds token limit:** The batch has too many slides. Split it into two smaller batches (add a new marker pair to the scaffold first).

**Slides are out of order:** The batch manifest wasn't followed. Check that each batch agent received only its assigned EX-N entries and generated slides in the specified order.
