Distill a technical documentation site into a learning path: $ARGUMENTS
1. Crawls the doc site and maps its hierarchy and conceptual layers
2. Prioritizes pages by conceptual importance, not doc ordering
3. Extracts key passages — core concepts, patterns, gotchas, design rationale
4. Builds a progressive learning path from foundations to advanced topics
5. Writes margin annotations connecting concepts across doc sections
6. Generates a markdown learning guide organized by conceptual progression
